Introduction

I'm certain you know how to add, subtract, multiply and divide. Maybe not very quickly, but you know a method. How about other operations? If I asked for √5 to 10 decimal places, could you do it by hand? How about ∛5? Or sin(5)? We often take it for granted that our calculators and computers can evaluate these functions, but how exactly? If we don't know a method by hand, then how can we get a computer to do it? I think it's worth giving some thought. As a bonus challenge, I'm going to go as far through this page as possible without using calculus. This will quickly become a big restriction.

n'th roots

Given a whole number n, and an input x, one simple but slow way to calculate the ⁿ√x is with the binary search algorithm. If x > 1, we know the answer will be larger than 0 and less than x. If x < 1, we know the answer will be greater than x and less than 1. That means we have an interval where the answer lies. ⁿ√x is an increasing function, meaning that a bigger input always means a bigger output. These are the conditions for binary search to work, we must know a finite interval where the answer lies, the function must be increasing everywhere in that interval, and the inverse of our function must be easy to calculate.
